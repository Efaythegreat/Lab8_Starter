## How are graceful degradation and service workers related?

Graceful degradation is like building an application with with all of its components first while slowly remove functionalities that may crash the application. While Service workers does this by running a standalone script that isn't apart of the web app, receiving requests and providing cached data and assets when the user is either offline or on slow connection so that it's always usable. It doesn't freeze or crash when there's a connection error like with graceful degradation, making it more reliable.
